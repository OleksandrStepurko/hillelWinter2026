# Технічне завдання (ТЗ)

## 1. Призначення та цілі

Сервіс призначений для автоматичного відстеження цільової директорії та
конвертації з'являючихся у ній файлів формату JSON у YAML і навпаки
(YAML у JSON). Сервіс веде детальне логування: вихідний файл,
результуючий файл, розмір вхідного/вихідного файлу, тривалість
конвертації, статус операції та діагностичні повідомлення.

## 2. Терміни та скорочення

-   **Сервіс** --- довготривалий процес (демон) спостереження за
    директорією та конвертації.
-   **Джерело** --- файл, виявлений у цільовій директорії для
    конвертації.
-   **Результат** --- файл після успішної конвертації.
-   **Валідатор** --- модуль перевірки коректності вмісту формату
    (JSON/YAML).

## 3. Область застосування

Сервіс запускається на сервері/робочій станції та працює цілодобово.
Підходить для інтеграцій, ETL-пайплайнів та DevOps-сценаріїв, де
потрібне дзеркалювання конфігурацій між JSON та YAML.

## 4. Обсяг робіт

1)  Розробка сервісу спостереження.
2)  Реалізація конвертації JSON→YAML і YAML→JSON.
3)  Валідація вхідних даних з інформативними помилками.
4)  Детальне логування операцій та помилок.
5)  Конфігурування через файл налаштувань та/або змінні середовища.
6)  Збірка, пакування (артефакт), базова документація та інструкції по
    деплою як служби (systemd/Windows Service за потреби).
7)  Набір автотестів (unit + інтеграційні) та тестові дані.
